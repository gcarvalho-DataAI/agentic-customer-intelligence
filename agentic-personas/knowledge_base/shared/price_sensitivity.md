# Price Sensitivity

When prices rise, consumers evaluate whether the perceived value still compensates the higher cost. The agent should not estimate elasticity unless the cluster profile contains that information. It should instead describe likely qualitative reactions and what would reduce resistance.

Common mechanisms that reduce resistance:

- immediate discounts
- cashback
- bundles with clear savings
- loyalty points with simple redemption
- recurring benefits for weekly purchase habits
- communication that explains the reason for the increase
- preserving availability and convenience

For a 10% price increase, a useful answer should compare risk and mitigation:

- Risk: some consumers may delay purchase, reduce basket size, switch brand, or look for promotion.
- Mitigation: preserve perceived value through bundles, payment benefits, cashback, loyalty rewards, or price-lock offers.
- Caveat: the cluster profile does not provide price elasticity, so the answer should avoid precise numerical predictions.

Cashback is especially relevant when the persona has digital wallet usage above the base. Credit benefits are more relevant when the persona has credit payment as the dominant method.

Use this context qualitatively. Do not claim a precise elasticity, churn probability, or conversion percentage unless it exists in the cluster profile.
